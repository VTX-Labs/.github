# Support

Here's how to get help with VTX Labs projects.

## Questions and usage help

- **[Discussions](https://github.com/orgs/VTX-Labs/discussions)** — the best
  place for questions, ideas, and general help.
- **Project README and docs** — start here; most common questions are answered
  in the project's own documentation.

## Found a bug or have a request?

Open an issue in the relevant repository using the appropriate template:

- **Bug report** — for reproducible problems.
- **Feature request** — for new ideas and improvements.
- **Documentation issue** — for unclear, missing, or incorrect docs.

Please search [existing issues](https://github.com/search?q=org%3AVTX-Labs&type=issues)
before opening a new one.

## Security issues

Do **not** use issues or discussions for security vulnerabilities. Follow the
private disclosure process in [SECURITY.md](SECURITY.md).

## Professional support

For priority support, custom development, security audits, or maintenance
retainers, contact [contact@vtxlabs.dev](mailto:contact@vtxlabs.dev) or visit
[vtxlabs.dev](https://vtxlabs.dev).
