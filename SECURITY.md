# Security Policy

Security is core to what VTX Labs does. We take vulnerabilities in our open-source
projects seriously and appreciate responsible disclosure.

## Reporting a vulnerability

**Please do not report security issues through public GitHub issues, discussions,
or pull requests.**

Instead, report privately via either:

1. **GitHub Security Advisories** — use the **"Report a vulnerability"** button
   under the **Security** tab of the affected repository (preferred).
2. **Email** — [contact@vtxlabs.dev](mailto:contact@vtxlabs.dev) with subject
   line `SECURITY: <repo name>`.

Please include:

- The affected repository and version.
- A description of the vulnerability and its impact.
- Steps to reproduce, ideally with a minimal proof-of-concept.
- Any suggested remediation, if you have one.

## What to expect

- **Acknowledgement** within **3 business days**.
- An assessment and severity rating, and a planned fix timeline.
- Credit in the advisory and release notes if you'd like it (or anonymity if you prefer).
- Coordinated disclosure — we'll agree on a public disclosure date with you.

## Scope

This policy covers code in repositories under the
[VTX-Labs](https://github.com/VTX-Labs) organization. Please only test against
your own instances and data — do not attack infrastructure or other users.

## Our commitment

We will not pursue legal action against good-faith security research that
respects this policy and applicable law.
